## 🎯 Key Features

### Story Mode Enhancements (V2.5)
Based on comprehensive tour guide research:

1. **70/30 Storytelling Rule**
   - 70% narrative/character-driven content
   - 30% facts embedded naturally
   - Maximum 3-5 key facts per tour

2. **Emotional Hooks**
   - First 15-30 seconds designed to capture attention
   - Universal emotions: fear, wonder, joy, mystery
   - Surprising facts that challenge assumptions

3. **Character-Conflict-Plot Framework**
   - Every narrative includes relatable character
   - Clear conflict/tension creating momentum
   - Satisfying resolution connecting to visitor

4. **Sensory Immersion**
   - Engages minimum 3 of 5 senses per tour
   - Target: 1 sensory detail per 100 words
   - Enhances memory retention by 40-60%

5. **Show Don't Tell**
   - Creates "Aha!" discovery moments
   - Lets listeners connect dots
   - Avoids spoon-feeding conclusions

6. **Deep-Dive Extended**
   - 1,200-1,800 words (8-12 min) vs. V2.0's 750-1,200 words
   - Multiple emotional beats and character spotlights
   - Optimal duration for sustained attention

### Classic Mode (V2.0 - Preserved)
- Flexible narrative beats
- Tour guide persona
- Balanced information mix
- Clear, informative style
- No changes to existing behavior

## 🔄 How It Works (End-to-End Flow)

### User Journey
1. **Settings Screen** (when UI toggle added):
   - User selects "Story Mode" or "Classic Mode"
   - Preference saved to Supabase `user_preferences.narrative_mode`
   - Falls back to AsyncStorage for offline access

2. **Tour Request**:
   - App loads user preferences from AppContext
   - AttractionInfoService includes `narrativeMode` in API request
   - Edge function receives preference

3. **Backend Processing**:
   - `promptGenerator.ts` checks `preferences.narrativeMode`
   - **If 'story-driven'**: Assembles 5 story blocks + story-focused task
   - **If 'fact-driven'**: Assembles V2.0 blocks (default behavior)
   - Returns appropriate prompt to AI provider

4. **AI Generation**:
   - OpenAI/Gemini receives mode-specific prompt
   - Generates narrative following mode guidelines
   - Returns audio/text to app

### Technical Flow
```
User Setting → PreferencesService → AppContext →
AttractionInfoService → Edge Function → promptGenerator →
Story-Driven Blocks OR Fact-Driven Blocks → AI Provider →
Generated Content
```

## 🐛 Troubleshooting

### Issue: Story mode not working
**Check**:
1. Database column exists: `narrative_mode` in `user_preferences`
2. Edge function deployed successfully
3. User preference saved correctly (check Supabase table)
4. Request includes `narrativeMode` field (check network logs)

### Issue: No difference between modes
**Check**:
1. Verify `narrativeMode` passed through entire request chain
2. Check backend logs for conditional logic execution
3. Compare actual prompts being sent to AI (add logging)
4. Ensure edge function redeployed after changes

### Issue: Preference not persisting
**Check**:
1. User logged in (Supabase requires auth)
2. AsyncStorage working for offline fallback
3. `PreferencesService.saveUserPreferences()` called
4. Database permissions correct (RLS policies)
